# Internationalization (i18n)

The Shalom Dev Portfolio supports 10 languages, providing a global reach for the developer's brand.

## Supported Languages
- **Kinyarwanda (`rw`)** - Default
- **English (`en`)**
- **French (`fr`)**
- **Swahili (`sw`)**
- **Arabic (`ar`)** - Supports RTL (Right-to-Left)
- **Spanish (`es`)**
- **German (`de`)**
- **Portuguese (`pt`)**
- **Chinese (`zh`)**
- **Hindi (`hi`)**

## Implementation Details

### `next-intl` Integration
We use `next-intl` for seamless integration with Next.js App Router.
- **Middleware:** `src/middleware.ts` handles locale detection and routing.
- **Routing:** `src/i18n/routing.ts` defines the supported locales and navigation wrappers.
- **Messages:** All content is stored in `client/messages/*.json`.

## Adding or Editing Content
To update content on the site, you must edit the respective keys in **all 10 JSON files** to ensure consistency.

### Example: Adding a Service
1. Open `client/messages/en.json`.
2. Add the new service to the `Services.items` object.
3. Repeat the step for `rw.json`, `fr.json`, etc.
4. The UI will automatically pick up the new data through the `t.raw()` mapping.

## RTL (Right-to-Left) Support
For Arabic (`ar`), the system automatically applies `dir="rtl"` in the main layout based on the current locale.
